UI Design

UI Design stands for user interface, and it contains the graphical layout of the application. This can consist on buttons, images, sliders, text entry field, searches and anything else the user would interact with.

In this project I wanted to expand and develop my own process when creating the UI components and overall design of an application. In my earlier projects I have focused on the UI system, however I was keen to delve deeper into creating a UI library to use to create the app/website for a aesthetically please user experience. In this project I will work on a UI library for an mobile app.

DESIGN PROCESS

Within the UI process, it is noticeable that their is no problem scoping which normally includes user research, interview

APP CONCEPT

The problem addressed within my app, is the difficulty users who create or produce music not being able to match with companies/organisation they are looking for to sell their music.

KAPE provides a platform that gives the music creator control over their vision.

USER PERSONA

LF WIREFRAME

With the creation of the user persona, I now can better understand the users interest and behaviour. Using this information I have created low-fidelity which the specific features that addresses the users needs.

1. Landing Page (demonstates the users registering/login process)

2. Profile (shows the different views avaliable in the profile tab).

3. Message (inbox and chat view)

4. Broadcast (shows how user can improve their visibility and successful collaboration)

5. Matches (demonstrates how the ads are shown to a user).

VISUAL LANGUAGE

Visual language is communication using visual elements. This can relate to the perception, comprehension and production of visible signs. There are a few considerations, the colour palette reflective not just the logo but the brand, creating a typography hierarchy which which dictates the tone of the information, a library of components, rules of animation for overall consistency.

To explore this, we need to use design inception worksheet. In this process, we want to clarify and understand a few things things. I have interview 5 people that have an interest in music / creating their own content which could use music.

      a) Why users would want to use this?
      b) What type of mood that would be associate with this paritcular idea?
      c) How would you communicate this mood through the visual communication?

MOOD BOARD

Using the design inception I've identified and group the two moods that can be further understood by using the mood board. The moodboard will help articular feeling associate with the concept, and the see which one fit the sort of overall concept. In this case I have two different user target groups. The first one is cool, creative and minimal coverying inspiration, and approachability relevant for building a using profile.

The second one is energtic, fast-pace and dynamic which suits the aim of enabling connections and collobration.

STYLE TILE

The moodboard is great to understand what colour platte is representative of the concept, the style tile looks at the colour platte in relation to the visual elements, a quick way to visual how this would work overall before moving into creating the prototype. The first style fits into the creative, and minimal style and there is a good balance of contrast to enable this to look cool.
