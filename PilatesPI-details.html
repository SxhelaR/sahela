REPUchain

UX Research

Problem

User wants to be able to see in real-time the score can change based on different information

Solution

Create a score feature, which breakdown the contributing factor the score. Includes notifications on real-time behaviour to improve score.

WHAT IS REPUchain?

REPUchain helps users to build a profile of who they are, to help support their applications for mortgages. It takes what a credit score does, and makes it transparency.

DESIGN PROCESS

COMPARATIVE ANALYSIS

The focus on this project, look at improve user experience in apps that already exist. I decide to place look at credit score apps, which are useful to look at when you want to seek a loan, mortgage etc. I will do a comparative analysis using the SWOT method on Experian and Clear Score, this would help evaluate the feature existing within the apps, to determine what opportunities or weakness that could improve the user experience, and repeat use of the feature.

SWOT ANALYSIS Experian ClearScore
Strenght Yes Yes
Weaknesses Yes Yes
Opportunity Yes Yes
Threats Mobile or Message Mobile or Message

RESEARCH

Based on the comparative analysis, I wanted to gather user’s opinions and thoughts on their experience. However, rather than conducting a survey, I believe I wouldn’t reach a wide range of users, which would also provide a range of honest feedback. Therefore, I decided to use social media to gather the feedback on both Experian and ClearScore customer. Whilst researching, I wanted to find concerns, frustrations on credit scores, and then more specific to the apps/ experience. This help outline the main difficulties, and what could potential be something I could improve. Below are some of the quotes, that provided from Twitter).

USER PERSONA

With the data gathered from the comparative analysis, and through research I identified two types of users from the target demographic.

AFFINITY MAP

With the research being mainly qualitative, I need to help make sense of the user research using the thematic process – this is why I used affinity map. Here I could begin to group the themes of the comments and identify patterns. The way I did this was writing out the comment on sticky notes, and placing them into a group. To finish of I would make a statement of what that group is identifying with. I did this also using Aurelius, to try this on online platform. I think it was good to use an online platform to see how this would work in team, when presenting the feedback.

PROBLEM STATEMENT

USER JOURNEY MAP

Next stage, I believe that the User Journey Map be very useful here, as the personas & the research will prevent false assumption. Using this method, we can visualise how the user personas would interact with application. The exercise allows me to completely empathise with the user, can bring to light an insight into the user point of view. This is important to understand before brainstorming and creating potential solutions. In this particular scenario, the user is seeking to apply for a mortgage, but to do that needs any app that will indicate a transparent view of their score with actions. The next aspect would be creating a list of touch points with a reference to motivations, channels, actions and paint-points. This all layout into a step-by-step interaction. Whilst doing, this I can see the “ups” and “downs” of the experience – highlighting areas of refinements. I can use a empathy map in conjunction to this.

USER FLOW

The User Journey map, has been useful to understand the potential pain points during the map, which could improve the experience whilst using the app. This now allows me to segment and define each screen and track what screen the users will typically see. The key aspect that are focused into the three area which the wireframes will at refining.

LOW-FIDELITY WIREFRAMES

I sketched out the score dashboard, without the transparency feature linked with the score. The individual can set goals, link with notification to provide reminder or alerts on behaviours.

USABILITY TESTING

One of the key elements I wanted to focus in one was the usability testing, in my previous project I had gained feedback, whereby I made iterative changes I was worked though. This time I want to observe the point of friction or error, take notes of the positive and negative comments. In the similar thematic process, categories, to understand what my priorities should be in my next iteration.

UI DESIGN

REFLECTION

For this project, I wanted to change the way I did problem scoping, to see a different way of uncovering a problem. I found both research and affinity map to be great tools to deal with qualitative information, and saw the importance of not forgetting about this type of data because it provides useful insight when translated into the user journey. I wanted to make sure that it was based on a true narrative hence, the statement created during the affinity map negated any false assumptions. This method of dealing with qualitative data, allow me to see a comparison with quantitative data. By relying on numbers, it gives an idea of what’s not working, however most often there is a dependency factor as to why it's not working, hence the qualitative data is relevant. As I have seen the qualitative data can differ from user to user, to find the theme you must group which is really useful. The second element was the reflection I did in my previous project where I wanted to expand my knowledge on useability testing. In this project, it was great to see the app, how it’s used and what that means. It definitely very different when you are selecting things to look at for, and how that mirrors what the user does. Usability testing, was important because it catches the gaps that maybe in earlier design processes wouldn’t catch. It a great comparison to check against the feedback on earlier attempts.
